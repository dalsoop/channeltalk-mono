# grounded-docs Philosophy — 근거에 갇힌·쉬운·절차 뼈대 문서 (SSOT)

이 팀은 **저장소/앱의 실제 코드·디렉터리 구조·해당 AGENTS.md·현재 대화맥락 '만'** 근거로 README·설명문서·해커톤 폼 답변을 쓰는 문서 작성 팀이다. 정본 믿음·채점축·게이트·반금지(지어냄·전문용어·글자수)를 담는다.
상위 정본: `references/agent-factory-philosophy.md`(공장 6축·Goodhart 방어). 이 문서는 그 도메인 확장이다.

## 믿음 (한 줄)

> 좋은 문서는 문장력이 아니라 **근거(코드·구조·AGENTS.md·대화맥락) 안에 갇혀 있고, 처음 보는 사람도 한 번에 이해되며, 절차 4요소 뼈대로 서 있고, 글자수 제약을 실측으로 지키는** 것이다.
> 그래서 이 팀은 writer(maker)만 두지 않는다. **근거·쉬움·구조/글자수 3 checker + 채점 스키마 + 게이트 루프까지** 갖추고, writer 가 자기 글에 도장 찍지 않는다(maker ≠ checker).

## 네 개의 축 (성공조건 = checker 채점축, 1:1)

| 축 | 무엇 | 방향·게이트 |
|---|---|---|
| **grounded_no_fabrication** | 실제 코드·구조·AGENTS.md·대화맥락에 근거 없는 지어낸 기능·수치·주장(문장 단위 대조) + 죽은 상대링크 | minimize · **==0** |
| **plain_language** | 쉬운 말 정도 0~10 — 설명 없이 던진 전문용어(maker/checker·pin·게이트·SSOT 등) 남발 0 + 처음 보는 심사자가 한 번에 이해됨 | maximize · **>=9.0** |
| **procedure_structure** | 절차 4요소(작동순서→지식근거→판단·채점기준→막힐 때)가 순서대로 뼈대로 존재 0~10 | maximize · **>=8.0** |
| **charcount_compliance** | 글자수 제약이 있을 때 공백 포함 `wc -m` 실측값이 상한 이내(제약 없으면 자동 통과). 초과 문자 수 | minimize · **==0** |

`pass = grounded==0 AND plain>=9 AND structure>=8 AND overflow==0`. 전부 실측(느낌 합격 없음).

## 절차 4요소 (문서의 뼈대 — 이 순서 고정)

1. **작동 순서** — 이걸 쓰면 무엇이 되는지 / 어떻게 동작하는지 순서대로. **결과 먼저**(정의·배경으로 열지 않는다).
2. **지식 근거** — 그 동작이 무엇(코드·구조·AGENTS.md 조항)에 근거하는지. 상대링크는 존재 확인된 경로만.
3. **판단·채점 기준** — 무엇을 만족하면 통과/맞다고 보는지(가능하면 실측 가능한 기준).
4. **막힐 때** — 안 될 때·주의·한계·폴백.

글자수를 줄여야 하면 세부를 덜되 **4요소는 유지**한다. README 는 처음 보는 사람 기준 친절한 첫인상(첫 화면에서 무엇에 쓰는지 바로 이해).

## 도메인 SSOT — 유일한 근거 정본

각 대상 저장소/앱의 **실제 코드·디렉터리 구조·해당 AGENTS.md·현재 대화맥락**이 유일한 사실 출처다. **별도 외부 정본 없음.** 근거에 없는 정보는 "근거 없음"으로 취급해 아예 쓰지 않는다(지어내기 금지). 검색으로 채우지 않는다.

## 안티패턴 — 'Docs tells' (찍으면 안 되는 신호)

| ID | Tell | 처방 |
|---|---|---|
| D-1 | 근거에 없는 기능·성능치를 그럴듯하게 단정 | 문장 단위로 근거 대조, 없으면 삭제/완화 [grounded] |
| D-2 | 실제 없는 파일로 상대링크를 검 | 존재 확인(`test -e`/`ls`)된 경로만 링크 [grounded] |
| D-3 | maker/checker·pin·게이트·SSOT 를 설명 없이 던짐 | 처음 보는 심사자 기준, 쓰려면 그 문장에서 괄호로 풀이 [plain] |
| D-4 | 정의·배경으로 열어 첫인상이 무거움 | 결과 먼저(무엇이 되는지)부터 [plain] |
| D-5 | 4요소 중 일부 누락·순서 뒤바뀜 | 작동순서→지식근거→판단기준→막힐 때 고정 [structure] |
| D-6 | 글자수 눈대중, 상한 초과 | 공백 포함 `wc -m` 실측, 초과 시 세부만 덜고 4요소 유지 [charcount] |
| D-7 | writer 가 자기 글을 스스로 합격 처리 | 신선 checker 3 이 실측 카운트/점수로 게이트 [maker≠checker] |

## Goodhart 방어 (점수를 정직하게)

1. **신선 checker 3** — writer 산출을 새로 읽어 각 렌즈로만 채점(자기 채점 금지).
2. **실측 + 결정적 선검사** — LLM 채점 전에 워크플로 코드가 죽은 링크·근거없는 수치·설명없는 전문용어·4요소 존재/순서·글자수 초과를 **직접** 잡는다. 각 축 = 근거·글자수는 `max(코드, 심판)`(엄격), 쉬움·구조는 코드 하한으로 눌러 후한 심판 방어. 출하 전 호출자가 `wc -m`(글자수)·`test -e`(상대링크)로 재확인.
3. **큰 재작성 감점** — 라운드 간 대량 변경은 표적 맞추기 의심.
4. **정직 > 칭찬** — 근거 없으면 "장인성 부족"이 아니라 문장:근거 로 짚는다. 못 쓰면 범위를 정직히 좁힌다(없는 정보는 안 쓴다).

## 종료 규칙

`stop = (전 게이트 pass) OR (round >= max_rounds=4) OR (2R 연속 개선 < 0.3)`. keep-best(best 에서만 채택), run ledger 에 라운드별 4축 수치와 clean 여부 기록. 무인 진행(루프 중 사람에게 안 묻는다).
